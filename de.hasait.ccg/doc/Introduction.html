Introduction - CommentCodeGen (CCG)
This is a eclipse plugin to generate code. Its special in the way you specifiy what code to generate.
The generated code is not placed in another file but generated within the file you are editing. This is done by adding a comment containing some keywords. These keywords are parsed by a builder and add a code block after that comment.
For example:
public class Folder {
   /*
    * My comments about the following property...
    * [c]<property name="file" type="File" />[/c]
    */
}
will be modified by the builder to:
public class Folder {
   /*
    * My comments about the following property...
    * [c]<property name="file" type="File" />[/c]
    * [bs]da0cefbcc0a8000a01652e619a0c7d76[/bs]
    */
   /**
    * Property file
    */
   public static final String PROPERTYNAME_FILE = "file";
   
   /** The single property field file. */
   private File _file;
   
   /**
    * Returns the file.
    */
   public final File getFile() {
      return _file;
   }
   
   /**
    * Sets the file.
    */
   public final void setFile(final File file) {
      _file = file;
   }
   // [be]da0cefbcc0a8000a01652e619a0c7d76[/be]
}
All the time you modify the above comment the generated code will be updated! So changing the name of the property or any other attribute will be reflected next time you save.
Modifying the above example (adding
many="true"
):
public class Folder {
   /*
    * My comments about the following property...
    * [c]<property name="file" type="File" many="true" />[/c]
    */
}
will be modified by the builder to:
public class Folder {
   /*
    * My comments about the following property...
    * [c]<property name="file" type="File" many="true" />[/c]
    * [bs]da0cefbcc0a8000a01652e619a0c7d76[/bs]
    */
   /**
    * Property file
    */
   public static final String PROPERTYNAME_FILE = "files";

   /** The many property field file. */
   private final java.util.List _files = new java.util.ArrayList();

   /**
    * Returns the array of all files.
    */
   public final File[] getFiles() {
      return (File[]) _files.toArray(new File[_files.size()]);
   }

   /**
    * Returns the file at the specified index.
    */
   public final File getFile(final int index) {
      return (File) _files.get(index);
   }

   /**
    * Returns if the specified file is contained.
    */
   public final boolean containsFile(final File file) {
      return _files.contains(file);
   }

   /**
    * Returns how many files are contained.
    */
   public final int fileSize() {
      return _files.size();
   }

   /**
    * Returns if file is empty.
    */
   public final boolean isFileEmpty() {
      return _files.isEmpty();
   }

   /**
    * Returns an {@link Iterator} for all files.
    */
   public final java.util.Iterator fileIterator() {
      return _files.iterator();
   }

   /**
    * Adds the specified file.
    */
   public final void addFile(final File file) {
      if (_files.contains(file)) {
         return;
      }
      _files.add(file);
   }

   /**
    * Removes the specified file.
    */
   public final void removeFile(final File file) {
      if (!_files.contains(file)) {
         return;
      }
      _files.remove(file);
   }

   /**
    * Remove all files.
    */
   public final void removeAllFiles() {
      File[] files;
      files = getFiles();
      for (int i = 0; i < files.length; i++) {
         removeFile(files[i]);
      }
   }
   // [be]da0cefbcc0a8000a01652e619a0c7d76[/be]
}
There are two extension-points:
  • ccgGenerators
    is for adding more generator-tags (like
    property
    )
  • ccgParsers
    is for parsing other comment syntaxes different from Java and C++
Available generators are:
  • includeFile

    Attributes are: path (string: workspace path)
    Nested elements: r
    Attributes of r are: search (string: text to find), replace (string: replace found text with this)
  • property

    Attributes are: name (string: property name), type (string: property type, e.g. String), many (boolean: single or multi value), threadsafe (boolean: adds correct synchronization), hooked (boolean: calls user-provided hook-methods on change), bound (boolean: fire change-events), otherend (string: property of set value referecing to this), otherendmany (boolean: otherend is single or multi value)
  • eventSupport
  • nop
Planned generators are:
  • delegate

    Generating method delegation for an instance. The code would not be a one time generated thing, but updated everytime the type of the instance changes to reflect correct up-to-date code.
